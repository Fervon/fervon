Por qué existe

Los agentes de IA son potentes. Sin control, son un riesgo.

Cada problema clásico de los agentes autónomos tiene una respuesta de ingeniería en Regenta.

📡

Cajas negras → Trazabilidad total

No sabes qué hacen ni qué archivos tocan. Regenta registra cada run, diff, log y decisión, con streaming SSE en vivo.

🛡️

Descontrol → Control Plane

Uno descontrolado puede borrar código o filtrar secretos. Regenta añade RBAC, file-locks, jaula de workspace y blocklist de comandos peligrosos.

🎯

Caos → Coordinator autónomo

Coordinar varios es caótico — se pisan los archivos. El Coordinator gestiona el desbloqueo de dependencias, locks exclusivos y retry automático.

🧠

Sin memoria → Cerebros persistentes

No entienden tu proyecto entre sesiones. Memoria por agente (brain v3, TF-IDF) que recuerda el contexto entre runs.

✋

Pérdida de control → Human-in-the-loop

Quieres delegar sin perder el control. Aprobación manual del CEO en cada fase crítica. Los deploys nunca son automáticos.

💸

Inferencia cara → Inferencia local

La inferencia cuesta dinero y escala mal. Modelos locales vía llama.cpp bajo demanda → coste $0, sin enviar tu código fuera.

Casos de uso

Mantén proyectos vivos, no scripts de una vez.

Le das objetivos y Regenta mantiene un backlog, reparte el trabajo entre la flota y lleva cada tarea de la idea al código revisado. Algunos de sus usos:

🚀

Construir features

Del objetivo en lenguaje natural al código: el scout analiza, el builder implementa bajo scope y el reviewer + QA validan.

♻️

Refactors con red de seguridad

allowedFiles acota el cambio, cada diff queda auditado y el reviewer comprueba los criterios de aceptación.

🧪

Tests y QA

El skill test-runner ejecuta la suite y captura fallos; el rol QA valida cobertura antes de aceptar.

🔍

Code review continuo

El reviewer analiza diffs en busca de bugs, problemas de seguridad y criterios — con tu aprobación final.

📦

Migraciones y bumps de deps

El scout evalúa el estado, se genera un plan por pasos, el builder ejecuta acotado y el reviewer valida.

🐛

Fix de bugs y mantenimiento

El scout localiza, el Coordinator asigna, el builder corrige y el reviewer aprueba — en bucle mientras el proyecto viva.

…y cualquier otra: si puedes describir la tarea y darle alcance, hay un agente para ello.

Control Plane

Seguridad por diseño

Delegar no significa perder el mando. Cada acción pasa por la guardia.

🔒

RBAC + Policies

Permisos granulares por rol: orchestrator, scout, builder, reviewer.

🎯

Scope allowedFiles

Cada tarea declara qué archivos puede tocar; el builder no se sale.

🔐

File Locks (TTL)

Adquisición exclusiva por tarea con expiración automática.

🧱

Path-guard

Jaula de workspace que resuelve symlinks con realpath, no solo léxico.

🚫

Terminal blocklist

Bloquea deploy/publish/login y prompts interactivos.

🗝️

Secret Vault

AES-256-GCM, IV por fila, master key fuera del repo (modo 0600).

📜

Audit Trail

Registro inmutable de cada acción, con stream SSE en /api/audit/stream.

🧯

Anti-loop + quarantine

Skip de tareas con 3+ fallos; keys con error aisladas sin reintentos en bucle.

🔐 Auth: JWT con expiración de 4h, bcrypt (cost 12), queries parametrizadas y rate-limiting. En producción la API exige JWT_SECRET — sin fallback inseguro.

FAQ

Preguntas frecuentes

¿Qué es Regenta?

La capa de control agentic: una sala de mando para dirigir una flota de agentes de IA con disciplina de ingeniería. No resuelve una tarea suelta — mantiene proyectos enteros vivos, llevando cada objetivo de la idea al código revisado con control total.

¿Cómo evita Regenta que un agente borre código o filtre secretos?

Por su Control Plane: RBAC por rol, scope allowedFiles por tarea, file-locks con TTL, jaula de workspace con path-guard, blocklist de comandos peligrosos y un secret vault cifrado con AES-256-GCM. Cada acción pasa por la guardia y queda en un audit trail inmutable.

¿Pierdo el control al delegar en la flota?

No. Regenta es human-in-the-loop por diseño: tú apruebas manualmente cada fase crítica y los deploys nunca son automáticos. Cada run, diff, log y decisión queda registrado, con streaming SSE en vivo.

¿Puedo crear mis propios agentes?

Sí. Defines tus propios agentes — nombre, rol, persona, modelo y skills — y los colocas en tu cadena de mando. Cada agente puede usar un provider y modelo distintos, o inferencia local.

¿Es Regenta gratis u open source?

No. Regenta es un producto comercial de Fervon, de código privado. Está en acceso anticipado: unirte a la lista fija condiciones early-bird; el precio se anuncia en el lanzamiento.

¿Cuánto cuesta la inferencia?

La licencia es de pago, pero la inferencia puede costar $0: Regenta puede despachar a modelos locales bajo demanda (vía llama.cpp) además de a 8 providers en la nube, sin enviar tu código fuera.
