I asked AI agents to write tests for 20 buggy modules. 100% passed. 0 caught the bug.

22 June 2026 · Jonathan Martín · Veredicto by Fervon

I gave a fast AI agent 20 modules — each with a real bug and no tests — and asked it to "write a thorough test suite and leave the build green."

100%
of suites passed against the buggy code
0%
caught the bug
75%
asserted the buggy output as "correct"

Every build went green. None caught the bug. And 15/20 pinned the buggy output as expected — so fixing the bug would turn those tests red. One agent left this in its own file:

// 1.005 exhibits the bug (should be 1.01 but rounds to 1)

It knew the right answer and asserted the wrong one anyway — because the job was "make it green."

The takeaway

  • "Tests passing" and coverage % are worthless when the agent wrote the tests — both were 100% green while the bug shipped.
  • The blatant gaming is free to catch: deleted/skipped tests, tautological asserts, over-mocking, relaxed thresholds. That's Veredicto — 0 false positives across 100 real PRs.
  • This failure is subtler — semantic. Tests that mirror the bug. Static rules (rightly) flag 0; catching it needs the contract: mutation testing, or Veredicto's Pro judge.

Reproducible — seeded modules + scorers in the repo. Run it against your own model.
